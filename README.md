# claude-code-boilerplate

Template for Claude Code projects with persistent memory, structured documentation, and behavioral rules.

## What's included

| File | Purpose |
|------|---------|
| `CLAUDE.md` | Operative instructions + Karpathy rules + references |
| `CLAUDE_MEMORY.md` | Current session state (gitignored, generated by setup.sh) |
| `docs/architecture.md` | Architecture, stack, infrastructure, I/O schema |
| `docs/lessons.md` | Operative lessons and established patterns |
| `.gitignore` | Includes CLAUDE_MEMORY.md + common patterns |
| `setup.sh` | Interactive initialization script |

## Usage

### 1. Clone or use as a GitHub template

```bash
git clone https://github.com/simonediroma/claude-code-boilerplate.git my-project
cd my-project
```

### 2. Run setup

```bash
chmod +x setup.sh
./setup.sh
```

The script asks for:
- Project name
- Invariant files that must never be touched

And automatically generates `CLAUDE_MEMORY.md` with today's date.

> **Working model:** each task is a separate session on a dedicated branch, auto-generated by Claude Code. There is no fixed branch — each session starts from `main` and opens its own PR.

### 3. Fill in `docs/architecture.md`

Fill in the sections with your project's architecture. This is the technical document Claude reads before every important decision.

### 4. Start Claude Code

```bash
claude
```

On first run, Claude will ask you to approve the `@docs/architecture.md` and `@docs/lessons.md` imports — approve both.

## Memory structure

```
CLAUDE.md            → fixed rules (committed)
CLAUDE_MEMORY.md     → session state (gitignored, updated by Claude)
docs/architecture.md → technical reference (committed, updated when architecture changes)
docs/lessons.md      → operative lessons (committed, grows over time)
```

pip install graphifyy && graphify install

## End of session

Claude automatically updates `CLAUDE_MEMORY.md` before the final commit with:
- Completed tasks
- Next steps
- Open blockers
- Current PR and branch
